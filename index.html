“Combining my love of music and programming, I created a music streaming and download/upload service with audio player visualization and the ability to share, play and create playlists. Using Postgress, I had 3 models, all many to many relationships. The visualization on the audio players used Web API, Audiocontext to analyze the track, thereafter plotting the data onto a HTML5 canvas to create the soundwave visual. Audiocontext was my biggest challenge. I had never used it before so was unable to predict the Cors restrictions I ran into. It was only realised on the final day of the project when I discovered that an audio node can only be connected to 5 audio sources at one time.”

Used: Sinatra framework, Ruby, JavaScript, psql, bcrypt, carrierwave, html5 canvas, web audio API, Audiocontext

“I Created this app as a solution to integrate, into our society, the many refugees arriving into our country as well as connecting those that want to help with those that need help. We used node.js as a framework, working in a team of three. After extensive planning, a colleague and I pair-coded tests for our back-end API using Mocha and Chai. I set up the API including authorization using bcrypt and JWT. I integrated the Stripe API into the app, assigning each donation to a particular refugee. For this, app it was necessary to have two different authenticated users, a charity and a potential host. The different roles on login was solved by having one user model, and storing the role, that was a hidden field on the registration form, into local storage through the payload on login and registartion. This method made it very easy to control accessibility on the front-end.”

Used: Node.js, JavaScript, TDD, Mocha, Chai, Google maps API, JWT, Bcrypt, Guardian API, mongoose, Ajax, JQuery, Bluebird
